**Introduction Session**

| This section covers everything the instructor should explain in the Introduction Video (Day 1). Students must watch and understand this before any coding begins. |
| :---- |

**1\. About the Company**

FirstCry Intellitots is a business operating in preschool admissions, daycare safety, early learning, parent communication, teacher workflows, child progress, centre operations, fees, and occupancy management. The team needs a clear digital system to manage day-to-day records, customer actions, staff ownership, and timely follow-ups.

For this internship project, students will study the real business flow and build a working prototype named AI Parent Meeting Summary Generator. The goal is to convert the given solution idea into a usable software reference implementation that supports the company team in practical operations.

**2\. The Problem They Face**

Every day, the company receives customer requests, operational updates, and follow-up requirements that need quick action.

| Problem: Teacher enters discussion notes; AI generates parent-friendly meeting summary and action points. At present, this type of work can become slow or scattered when it depends only on calls, WhatsApp messages, spreadsheets, or manual follow-up. |
| :---- |

This results in:

\- Wasted time for the team

\- Slow response to important customers or priority cases

\- Missed revenue opportunities because follow-ups are delayed

\- No clean system to track progress, ownership, and history

**3\. What We Are Building**

We are building AI Parent Meeting Summary Generator \- a smart working prototype that solves this problem end to end.

How the system works:

\- Step 1 \- Parent/Guardian/Student enters or updates information through the parent enquiry form, teacher dashboard, centre admin panel, daycare routine entry screen, classroom activity screen, parent portal, or counsellor follow-up screen

\- Step 2 \- Backend APIs validate the input and save it with source, status, owner, and timestamp

\- Step 3 \- The system applies parent communication workflow that captures feedback, schedules discussions, generates notices, and tracks parent satisfaction signals

\- Step 4 \- AI or rule-based logic prepares useful outputs such as recommendations, summaries, alerts, or messages

\- Step 5 \- Teacher/Admin/Centre Head/Counsellor views records, status, priorities, and action history in the dashboard

\- Step 6 \- Automated reminders, confirmations, or follow-up messages are generated where required

\- Step 7 \- The final deployed prototype is tested, documented, and presented with report, PPT, GitHub, and demo video

**4\. Technology Stack**

The following technologies will be used in this prototype:

| Layer | Technology | Role in This Project |
| :---- | :---- | :---- |
| Frontend | HTML, CSS, JavaScript / React | Parent meeting scheduler, demo feedback form, sentiment dashboard, notice generator screen, parent message preview, and communication history |
| Backend | Node.js Express or Python Flask | Meeting slot APIs, feedback collection endpoints, sentiment aggregation logic, notice template storage, parent communication endpoints, and reports |
| AI Layer | OpenAI API / Gemini API or rule-based prompts | Generate parent meeting summaries, notices, WhatsApp messages, sentiment insights, and professional follow-up communication |
| Database | MySQL / PostgreSQL / Firebase | Store children, parents, admissions, enquiries, classrooms, attendance, routines, allergies, meals, photos, lesson plans, curriculum activities, milestones, teacher tasks, feedback, fees, occupancy, transport, supplies, and communication history |
| Messaging/Reports | WhatsApp Business API / Email API / PDF or CSV export | Send confirmations, reminders, alerts, summaries, and admin reports |
